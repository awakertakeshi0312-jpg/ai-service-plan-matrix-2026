Executive briefing

LLMツールの実力を
「任せられる仕事」で見る

最新AIツールの差は、会話の自然さだけでは測れない。重要なのは、調査、判断補助、制作、コード修正、ローカル実行、自動化まで、どの仕事をどのリスクで委任できるかである。

委任レベルを見る ツール分類を見る 実演シナリオを見る

人間

Conclusion

モデル名の順位より、業務への接続度が価値を決める

2026年時点の勝ち筋は、すべてを単一モデルに任せることではない。目的別にツールを組み合わせ、重要判断だけ人間が握る設計である。

Insight 01

LLMは「答える」から「作業する」へ移った

Deep Research、Coding Agent、ブラウザ操作、ファイル編集、RAGにより、成果物まで作る流れが標準化している。

Insight 02

ローカルLLMは秘密データと低コスト反復に強い

Qwen、Gemma、gpt-oss、Phiなどにより、端末内要約、社内文書処理、開発補助の実用ラインが上がった。

Insight 03

エージェント化の核心は権限設計

AIが実行できるほど価値は増えるが、誤実行の影響も大きくなる。承認、ログ、ロールバックが競争力になる。

Delegation ladder

AIツールの実力は「委任レベル」で説明する

非エンジニアに伝えるときは、IQのような抽象指標ではなく「どの仕事を任せられるか」に翻訳する。

Lv1相談

相談相手

質問回答、翻訳、要約、文章案、壁打ち。最も安全で導入しやすいが、成果物の責任は人間に残る。

人間の役割: 目的を伝える、内容を確認する
Lv2調査

調査員

公開情報、PDF、社内資料、会議ログを読み、論点・比較・出典つきメモにまとめる。

人間の役割: 情報源、前提、結論の妥当性を見る
Lv3制作

実務アシスタント

企画書、提案書、スライド、HTML、表、画像案、コード案を作る。人間は編集長として品質を上げる。

人間の役割: 採否判断、ブランド調整、最終表現
Lv4実行

作業者

ブラウザ、CLI、GitHub、ローカルファイル、テストを操作し、実際に修正・検証する。

人間の役割: 権限管理、レビュー、戻せる状態の維持
Lv5運用

小さな部署

定期調査、レポート生成、問い合わせ処理、開発保守、ナレッジ更新を複数ツールで回す。

人間の役割: 目的設定、KPI、例外処理、責任分界

Tool landscape

主要ツール群と、見せるべき実力

同じAIでも「会話」「調査」「開発」「ローカル」「生成」「自動化」で評価軸は変わる。

汎用LLM

会話、分析、文章化、判断補助の中核。

  • 役員説明、顧客文面、企画の初稿
  • 曖昧な相談を論点化
  • 人間の思考速度を上げる

調査AI

Web、PDF、社内資料を読み、出典つきで要約する。

  • 市場調査、競合比較、技術調査
  • 長文資料から意思決定メモ
  • 情報の再利用性を高める

開発AI

コードを読み、修正し、テストし、差分を説明する。

  • バグ修正、UI改善、PR対応
  • レポジトリ全体の文脈理解
  • 開発者の作業単位を拡張

ローカルLLM

PCや自前サーバーで動き、機密データと反復処理に強い。

  • 社内文書要約、ローカルRAG
  • Qwen、Gemma、gpt-oss、Phi系
  • コストとプライバシーの制御

画像・動画AI

広告、SNS、研修、説明素材を短時間で試作する。

  • ビジュアルコンセプトの量産
  • ストーリーボード、素材案
  • ブランド監修が品質を決める

自動化AI

定期処理、通知、データ連携、ワークフローを動かす。

  • 毎朝レポート、問い合わせ整理
  • 人間承認つきの実行フロー
  • ログと権限設計が必須

Model tier framework

クラウドLLMとローカルLLMのTier評価表

Tierは「絶対的な頭の良さ」ではなく、商用利用でどこまで任せられるかの実務評価。Cloudは実行品質、Localは実行品質に加えてハード要件と機密性で見る。

表示は5段階、内部評価は10点

読み手にはTier 1-5で直感的に見せ、実際の診断では品質、速度、コスト、再現性、リスク管理を10点満点で採点する。

評価軸

1. 成果物品質 2. 推論/コーディング 3. ツール実行 4. 速度/コスト 5. 長文/RAG 6. 機密性 7. 運用しやすさ。

Cloud LLM Tier

クラウドLLMは、最高品質の判断、エージェント実行、顧客向け成果物、長文推論に強い。機密性と継続コストは設計で管理する。

Operating model

商用導入の実装モデル

安定した導入は、クラウドAI、ローカルAI、人間承認を分けて設計するところから始まる。

Cloud AI

  • 最高品質の推論と文章化
  • 外部情報を含む深い調査
  • 難しい判断のセカンドオピニオン
Human Control Plane

目的、権限、承認、ログ、評価指標を人間が握る。AIには作業単位を渡す。

Local AI

  • 機密文書の要約と分類
  • 低コストな反復処理
  • 社内ナレッジ検索とRAG

Decision guide

どのAIから始めるべきか

技術選定は「流行」ではなく、データの機密性、反復頻度、失敗時の影響で決める。

外部向け品質が最優先

クラウド上位LLMを主軸にし、人間が編集長として最終品質を担保する。

機密データが多い

ローカルLLMと社内RAGを先に整備し、外部APIへ出す情報を制限する。

開発速度を上げたい

開発AIに小さな修正、テスト、レビューコメント対応から任せる。

毎日同じ作業がある

自動化AIで定期レポート、通知、分類、下書き生成をワークフロー化する。
